  1. Master Android's Sensors

    Varity of Android sensor types listed by Android Authority, which include the following:

    Environmental Sensors measuring conditions such as:

    • Air Temperature
    • Pressure
    • Humidity
    • Ambient Light Levels

    Sensors measuring a device's physical position, like:
    • Proximity to Objects
    • Geomagnetic Field Detection

    Sensors mesuring a device's motion, including:
    • Accelerometers
    • Gravity Sensors
    • Guroscopes
    • Rotation Vector Sensors

    These sensors can either be hardware-based, where physical components are built into the device for the purpose of each sensor, or software-based, which collects data from multiple sensors from the hardware. This source also cover examples on how to communicate with each sensor and their respective listeners.

  2. Motion Sensors in iOS

    Motion sensors in iOS devices function rather differently than Andriod devices, but still perform a lot of the same tasks at about the same level as its counterpart. In fact, the hardware can be so percise as to create a Virtual Reality experience for some users. Most iOS devices are equipped with three sensors:

    • Accelerometer. Measures non-gravitational acceleration along the X, Y or Z axes.
    • Gyroscope. Measures device rotation with respect to Earth gravity.
    • Magnetometer. Measures the strength of the magnetic field surrounding the device.

    All three of these sensors are handled through the CMMotionManager class. This makes programming in this environment much simpler when handling sensor projects.

  3. How to Build an Application to Test Motion Sensors in Android

    This article walks their reader through six steps on how to test their Android motion sensors. The specific components that will be tested in this case will be Accelerometer and Gyroscope. Here are the ordered steps to change the mobile device's screen color as it's moved around in different situations:

    1. Create a New Project in Android Studio with Java as the programming language.
    2. Add features in the AndroidMainfest.xml file to add the requirement of the physical sensors needed for this project.
    3. Add code to the activity_main.xml file to update the output formatting.
    4. Add functions to the Accelerometer.java file to add the listeners to for the Accelerometer and methods for responses to motion engagement.
    5. Add functions to the Gyroscope.java file to add the listeners to for the Gyroscope and methods for responses to motion engagement.
    6. Add functions to the MainActivity.java file to add the actions to carry out in reponse to engagement of the Accelerometer.java and Gyroscope.java files.
